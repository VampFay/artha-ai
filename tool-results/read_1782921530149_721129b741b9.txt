     1→[Heading 1] FinSight AI — Detailed Implementation Plan
     2→[Heading 2] Project Title
     3→[First Paragraph] FinSight AI: AI-Powered Tax Readiness and Financial Health Assistant
     4→[Heading 2] Core Idea
     5→[First Paragraph] FinSight AI is a privacy-first web application that helps users organize financial documents, estimate tax readiness, analyze spending behavior, understand financial health, and generate CA-ready summary reports.
     6→[Body Text] The project does not aim to replace a Chartered Accountant or investment adviser. Instead, it acts as a smart assistant that makes financial data easier to understand, verify, and prepare.
     7→[Body Text] A major focus of this project is UI/UX quality. Since finance and tax data can feel confusing, the interface must be simple, clean, trustworthy, and easy to use. The system should not only work correctly, but also feel professional and comfortable for real users.
     8→[Heading 1] 1. Project Priorities
     9→[First Paragraph] The project will be built around five major priorities:
    10→[Heading 2] 1. Functional correctness
    11→[First Paragraph] The system should correctly extract, organize, calculate, and display financial information.
    12→[Heading 2] 2. UI/UX quality
    13→[First Paragraph] The application should be easy to navigate, visually clean, and understandable even for users with low financial knowledge.
    14→[Heading 2] 3. Privacy and trust
    15→[First Paragraph] The system should clearly show consent, data usage, masking of sensitive information, and deletion controls.
    16→[Heading 2] 4. Performance and optimization
    17→[First Paragraph] Every module must be efficient. No unnecessary database queries, no repeated calculations, no oversized components, and no wasteful processing.
    18→[Heading 2] 5. Clean code discipline
    19→[First Paragraph] There must be no dead code, unused functions, unused imports, redundant logic, unnecessary files, duplicate components, or inefficient algorithms.
    20→[Body Text] Every line of code should have a purpose.

   108→[Heading 1] 5. UI/UX Design Philosophy
   109→[First Paragraph] UI/UX is a core part of this project.
   110→[Body Text] The user should never feel lost. Every screen must clearly answer:
   111→[Source Code] Where am I?
   112→What do I need to do?
   113→What has the system understood?
   114→What is missing?
   115→What should I verify?
   116→What can I download?
   117→[Heading 2] Design principles
   118→[Heading 3] 5.1 Simplicity
   119→[First Paragraph] Finance and tax are already complicated. The UI must reduce complexity, not add to it.
   120→[Body Text] Use:
   121→[Compact] Simple language.
   122→[Compact] Short labels.
   123→[Compact] Clear explanations.
   124→[Compact] Visual indicators.
   125→[Compact] Step-by-step flows.
   126→[First Paragraph] Avoid:
   127→[Compact] Too many buttons.
   128→[Compact] Dense tables without summaries.
   129→[Compact] Unexplained financial terms.
   130→[Compact] Overloaded dashboards.
   131→[Compact] Long forms without grouping.
   132→[Heading 3] 5.2 Trust
   133→[First Paragraph] Since users are uploading financial documents, the UI must feel secure.
   134→[Body Text] Add:
   135→[Compact] Consent screen.
   136→[Compact] Data masking.
   137→[Compact] “Delete document” option.
   138→[Compact] “Why we need this” tooltips.
   139→[Compact] Processing status.
   140→[Compact] Confidence scores.
   141→[Compact] Clear disclaimers.
   142→[Heading 3] 5.3 Explainability
   143→[First Paragraph] Do not just show numbers. Explain them.
   144→[Body Text] Example:
   145→[Body Text] Instead of only showing:
   146→[Source Code] Tax Readiness Score: 68
   147→[First Paragraph] Show:
   148→[Source Code] Tax Readiness Score: 68/100
   149→
   150→Your score is reduced because rent receipts and insurance proof are missing.
   151→[Heading 3] 5.4 Progressive disclosure
   152→[First Paragraph] Do not show everything at once.
   153→[Body Text] Show summary first, details later.
   154→[Body Text] Example:
   155→[Source Code] Financial Health Score: 74/100
   156→Main issue: Emergency fund is low.
   157→View Details
   158→[Heading 3] 5.5 Mobile-friendly design
   159→[First Paragraph] The app should work on laptops and mobile screens.
   160→[Body Text] Important pages must be responsive:
   161→[Compact] Login.
   162→[Compact] Dashboard.
