     4→[Heading 1] FinSight AI — Detailed Implementation Plan
     5→[Heading 2] Project Title
     6→[First Paragraph] FinSight AI: AI-Powered Tax Readiness and Financial Health Assistant
     7→[Heading 2] Core Idea
     8→[First Paragraph] FinSight AI is a privacy-first web application that helps users organize financial documents, estimate tax readiness, analyze spending behavior, understand financial health, and generate CA-ready summary reports.
     9→[Body Text] The project does not aim to replace a Chartered Accountant or investment adviser. Instead, it acts as a smart assistant that makes financial data easier to understand, verify, and prepare.
    10→[Body Text] A major focus of this project is UI/UX quality. Since finance and tax data can feel confusing, the interface must be simple, clean, trustworthy, and easy to use. The system should not only work correctly, but also feel professional and comfortable for real users.
    11→[Heading 1] 1. Project Priorities
    12→[First Paragraph] The project will be built around five major priorities:
    13→[Heading 2] 1. Functional correctness
    14→[First Paragraph] The system should correctly extract, organize, calculate, and display financial information.
    15→[Heading 2] 2. UI/UX quality
    16→[First Paragraph] The application should be easy to navigate, visually clean, and understandable even for users with low financial knowledge.
    17→[Heading 2] 3. Privacy and trust
    18→[First Paragraph] The system should clearly show consent, data usage, masking of sensitive information, and deletion controls.
    19→[Heading 2] 4. Performance and optimization
    20→[First Paragraph] Every module must be efficient. No unnecessary database queries, no repeated calculations, no oversized components, and no wasteful processing.
    21→[Heading 2] 5. Clean code discipline
    22→[First Paragraph] There must be no dead code, unused functions, unused imports, redundant logic, unnecessary files, duplicate components, or inefficient algorithms.
    23→[Body Text] Every line of code should have a purpose.
